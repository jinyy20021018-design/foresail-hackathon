ForeSail 用户使用指南

本指南面向贸易运营人员，帮助你在 ForeSail 中完成 Case 创建、文档审核、Agent 监控、 风险识别与行动跟进。无需了解后端技术细节，按页面操作即可完成 Demo 与日常工作流。

适用版本：MVP 3.0 前端地址：http://127.0.0.1:5173 语言：简体中文 English

1. ForeSail 是什么？

ForeSail 是一个贸易 Case 监控工作台。你可以把每一票货运/贸易业务当作一个 Case， 系统会帮你：

读懂贸易文档

上传合同、订舱确认书、信用证等文件，自动抽取关键字段并展示证据原文。

建立监控画像

根据船名、港口、航线和截止日期，确定这票货需要关注什么。

扫描外部事件

识别船期延误、港口中断、天气等外部信息，并判断是否与当前 Case 相关。

输出行动建议

生成风险摘要、义务提醒、推荐行动与处置方案，支持人工确认后继续监控。

你可以把它理解为：围绕单笔贸易 Case，从“文档 → 事实确认 → 外部监控 → 风险行动”的一站式工作台。

2. 如何启动系统

ForeSail 由前端页面和后端 API 两部分组成，需同时运行。

启动后端

在终端进入 backend 目录后执行：

.\.venv\Scripts\Activate.ps1
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000

启动前端

另开一个终端，进入 frontend 目录后执行：

npm run dev

打开应用

浏览器访问：http://127.0.0.1:5173

首次使用提示：若后端尚未创建虚拟环境，需先执行 python -m venv .venv 与 pip install -r requirements.txt。 若希望 Agent 摘要或文档抽取使用 LLM，请在 backend/.env 中配置 OpenAI API Key；不配置也可运行 Demo。

3. 界面与导航

打开 ForeSail 后，你会看到以下主要区域：

区域 作用
左侧边栏 品牌入口、Case Library 等导航，以及今日概览快照（At Risk、Action Required 等统计）。
顶部栏 中英文切换、当前用户信息。
主工作区 Case Library、新建 Case 或单个 Case 的详细工作台。

常用页面路径：

  • Case Library — 查看全部 Case
  • Create New Case — 上传文档并创建新 Case
  • Case Workspace — 进入某个 Case 的详细操作页面

4. Case Library 案例库

Case Library 是你管理所有贸易 Case 的入口。

你能做什么

  • 查看 Case 总数、At Risk、Action Required、7 天内截止日、信息缺口等汇总卡片
  • 按 Case ID、船名、航线、客户等关键词搜索
  • 按状态、风险等级、负责人筛选
  • 点击 Create New Case 创建真实业务 Case
  • 点击 Clean Demo 或 Conflict Demo 快速体验系统

列表字段说明

字段 含义
Case IDCase 唯一编号
Shipment / Vessel船舶与货运信息
Route航线
Status当前 Case 状态
Risk Level风险等级（High / Medium / Low）
Next Deadline最近的关键截止日
Open Actions待处理行动数量
Info Gaps尚未补齐的信息缺口
Conflicts未解决的字段冲突
快速体验建议：第一次使用时，可直接点击 Clean Demo 进入完整流程；若想体验字段冲突处理，可选择 Conflict Demo。

5. 新建 Case

通过 Create New Case 页面，你可以从真实贸易文档开始建立一个 Case。

  1. 准备并选择文档 建议至少上传 Contract / PO；推荐同时上传 Booking Confirmation 与 Letter of Credit。
  2. 创建 Draft Case 系统会先创建草稿 Case，再逐个上传文档。
  3. 运行文档抽取 点击抽取后，系统会从文档中提取买方、卖方、船名、港口、ETD/ETA、付款方式等字段。
  4. 查看抽取 Trace 页面会显示“创建 Case → 上传 → 抽取 → 检测冲突 → 待人工审核”等步骤进度。
  5. 人工审核字段 检查自动填充结果，必要时手动修改；可查看字段对应的证据原文。
  6. 进入 Case Workspace 审核完成后进入该 Case 的详细工作台，继续确认字段与运行 Agent。

支持的文档类型

文档是否建议上传常见抽取字段
Contract / PO必填买方、卖方、商品、金额、贸易术语、付款方式
Booking Confirmation推荐船名、装港、卸港、ETD、ETA
Letter of Credit推荐金额、币种、最迟装运日、LC 到期日、付款方式
Insurance Certificate可选保单号、保险类型
Other Supporting Documents可选其他辅助信息

6. Case Workspace 工作台

进入某个 Case 后，你会看到该 Case 的全部操作面板。顶部有两个关键按钮：

  • Run Agent Monitoring Cycle — 运行一轮 Agent 监控
  • Continue Monitoring — 在需要行动时确认后继续监控
若存在未解决的高严重度字段冲突，系统会阻止你确认字段或运行 Agent。请先到 Conflicts 页签处理。

九个功能页签

页签你会看到什么典型操作
Overview Case 快照、监控画像、最新 Agent 运行、状态时间线 快速了解当前 Case 全貌
Documents & Evidence 已上传文档、抽取字段、证据片段 上传文档、抽取字段、批准/编辑/拒绝字段，点击 Confirm Fields
Conflicts 不同文档之间的字段冲突 查看并解决 High 级冲突
Agent Runs Agent 运行总结、历史记录、逐步 Trace 查看 Agent 做了什么、为何得出当前结论
External Events 外部事件列表与采集配置 查看船期、港口、天气等事件是否影响本 Case
Risks & Obligations 风险摘要、义务与截止日、信息缺口 判断是否需要升级处理
Actions & Drafts 推荐行动与行动草稿 查看建议联系谁、做什么、何时完成
Treatment Plans 风险处置方案与审批包 为已识别风险制定处理方案
Audit Agent 运行历史与审计信息 追溯历史决策与系统输出

Workflow Stepper 工作流进度

工作台顶部的 Workflow Stepper 会告诉你当前 Case 处于哪一步，例如：

  • 文档是否已上传
  • 字段是否已抽取
  • 冲突是否已解决
  • Case 事实是否已确认
  • Agent 是否已运行

建议始终先让 Workflow Stepper 走到“可运行 Agent”的状态，再执行监控。

7. 运行 Agent 监控

Agent 相当于替你完成一轮“外部信息值班检查”。你只需点击按钮，系统会自动完成后续分析。

运行前请确认

  • 已在 Documents & Evidence 中点击 Confirm Fields
  • 不存在未解决的 High 级字段冲突
  • Case 当前不是已在持续监控中的 MONITORING 状态

Agent 会为你做什么

  1. 读取 Case 与已确认事实 — 以你确认过的贸易信息为准。
  2. 生成监控画像 — 明确要盯哪些船、港口、航线区域和截止日。
  3. 采集外部事件 — 默认使用 Mock 事件；若配置了真实数据源，也会尝试拉取新闻与天气。
  4. 判断事件相关性 — 将事件分为 Relevant（相关）、Watch（观察）、Irrelevant（无关）。
  5. 映射风险敞口 — 如运输、LC 截止日、港口作业、付款时间线等。
  6. 生成义务、缺口与行动 — 告诉你该做什么、缺什么信息。
  7. 更新 Case 状态并输出总结 — 在 Agent Runs 页签查看 Summary 与 Trace。

运行后你应该看哪里

页签关注内容
OverviewCase 状态是否变为 AT_RISK 或 ACTION_REQUIRED
Agent RunsAgent 总结、扫描事件数、相关事件数、执行步骤
External Events哪些外部事件被纳入分析
Risks & Obligations触发了哪些风险、有哪些截止日
Actions & Drafts推荐联系承运人、复核 LC、通知财务等行动
当 Case 进入 ACTION_REQUIRED 后，请查看行动板并点击 Continue Monitoring，Case 会回到 MONITORING 持续监控状态。

8. Case 状态说明

ForeSail 用状态反映 Case 当前处理阶段：

DRAFT 草稿 ACTIVE 已激活 WATCHING 监控中 AT_RISK 存在风险 ACTION_REQUIRED 需要行动 MONITORING 持续监控
状态对你意味着什么
DRAFTCase 刚创建，文档或字段尚未准备完成。
ACTIVE核心贸易字段已确认，可以开始监控。
WATCHING系统正在持续观察外部事件。
AT_RISK检测到与 Case 相关的实质性风险事件。
ACTION_REQUIRED系统已生成推荐行动，等待你确认处理。
MONITORING你已确认行动，Case 回到持续监控。

9. Demo 完整演示（CAPEMOLLINI）

系统内置 Demo Case，适合第一次体验完整流程。

Demo 背景

  • 船舶：CAPEMOLLINI
  • 航线：上海 → 吉大港 → 达卡
  • 付款方式：LC at sight
  • 贸易术语：CIF
  1. 进入 Case Library，点击 Clean Demo 创建演示 Case。
  2. 打开 Case Workspace，在 Documents 页签查看或补充 Contract / Booking / LC。
  3. 抽取并确认字段，确保 Case 事实准确。
  4. 点击 Run Agent Monitoring Cycle，等待 Agent 完成分析。
  5. 查看 External Events，例如“CAPEMOLLINI ETA 延误 5 天”可能被标为相关。
  6. 查看 Risks & Actions，系统可能建议联系承运人、复核 LC 装运时间等。
  7. 点击 Continue Monitoring，完成从风险识别到持续监控的闭环。
若你想体验“文档字段冲突必须先处理”的场景，可在 Case Library 中创建 Conflict Demo。

10. 常见问题

页面打不开或数据加载失败？

请确认后端是否已启动，并访问 http://127.0.0.1:8000/api/health 检查是否返回 {"status":"ok"}。

为什么无法运行 Agent？

  • 尚未点击 Confirm Fields
  • 存在未解决的 High 级字段冲突
  • Case 已处于 MONITORING 状态

为什么看不到 LLM 摘要？

需要在 backend/.env 中配置 OPENAI_API_KEY，并启用 USE_LLM_SUMMARY=true。未配置时系统会使用确定性规则生成摘要，Demo 仍可正常运行。

外部事件是真实的吗？

默认 workflow 使用已配置的真实事件源。Overview 顶部会显示 Event Mode（REAL / HYBRID / MOCK），方便用户确认当前事件来源。

我之前创建的 Case 不见了？

Case 数据保存在本地数据库与浏览器缓存中。若更换目录、清空浏览器 localStorage 或使用新数据库文件，历史 Case 可能不会自动保留。

安全提醒：请勿将包含 API Key 的 .env 文件分享给他人或提交到 Git 仓库。
