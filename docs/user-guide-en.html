1. What is ForeSail?

ForeSail is a trade case monitoring workbench. Each shipment or trade deal is managed as a Case. The system helps you:

Read trade documents

Upload contracts, booking confirmations, and letters of credit. Key fields are extracted with evidence snippets.

Build a watch profile

Define what to monitor based on vessel, ports, route, and key deadlines.

Scan external events

Detect delays, port disruptions, weather, and other signals that may affect your case.

Recommend actions

Surface risk summaries, obligations, recommended actions, and treatment plans for human review.

In short: ForeSail is a one-stop workspace for a single trade case—from documents to confirmed facts, external monitoring, and actionable risk response.

2. How to Start the System

ForeSail has a frontend UI and a backend API. Both must be running.

Start the backend

In a terminal, go to the backend folder and run:

.\.venv\Scripts\Activate.ps1
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000

Start the frontend

In a second terminal, go to the frontend folder and run:

npm run dev

Open the app

Visit http://127.0.0.1:5173 in your browser.

First-time setup: If the virtual environment does not exist yet, run python -m venv .venv and pip install -r requirements.txt first. To enable LLM summaries or smarter document extraction, configure OPENAI_API_KEY in backend/.env. The demo flow works without it.

5. Create a New Case

Use Create New Case to start from real trade documents.

  1. Select documents Upload at least a Contract / PO. Booking Confirmation and Letter of Credit are strongly recommended.
  2. Create a draft case The system creates a draft case first, then uploads each document.
  3. Run extraction ForeSail extracts buyer, seller, vessel, ports, ETD/ETA, payment terms, and more.
  4. Review the extraction trace Progress steps appear: create case → upload → extract → detect conflicts → ready for review.
  5. Review fields manually Check autofill results, edit if needed, and inspect evidence snippets.
  6. Enter Case Workspace Continue with field confirmation and agent monitoring.

7. Run Agent Monitoring

The agent performs one round of external-information watch duty for you. Click the button—the system handles the analysis.

Before you run

  • You have clicked Confirm Fields under Documents & Evidence
  • No unresolved high-severity conflicts remain
  • The case is not already in MONITORING status

What the agent does

  1. Load the case and confirmed facts — uses the trade information you approved.
  2. Build a watch profile — defines vessel, ports, route regions, and deadlines to watch.
  3. Fetch external events — uses configured real news/weather sources; mock events are only used when explicitly selected by an administrator.
  4. Score event relevance — labels events as Relevant, Watch, or Irrelevant.
  5. Map risk exposures — shipping, LC deadlines, port operations, payment timeline, etc.
  6. Generate obligations, gaps, and actions — tells you what to do and what is still missing.
  7. Update case status and summary — review results under Agent Runs.

9. Full Demo Walkthrough (CAPEMOLLINI)

ForeSail includes a built-in demo case for a complete first-run experience.

Demo background

  • Vessel: CAPEMOLLINI
  • Route: Shanghai → Chittagong → Dhaka
  • Payment: LC at sight
  • Incoterm: CIF
  1. Open Case Library and click Clean Demo.
  2. Open Case Workspace and review Contract / Booking / LC under Documents.
  3. Extract and confirm fields so case facts are accurate.
  4. Click Run Agent Monitoring Cycle and wait for analysis to finish.
  5. Check External Events — e.g. “CAPEMOLLINI ETA delayed by 5 days” may be marked Relevant.
  6. Review Risks & Actions — suggestions may include contact carrier or prepare LC amendment.
  7. Click Continue Monitoring to close the loop from risk detection to ongoing watch.

10. FAQ

The page won't load or data fails to fetch?

Make sure the backend is running and visit http://127.0.0.1:8000/api/health. You should see {"status":"ok"}.

Why can't I run the agent?

  • You have not clicked Confirm Fields
  • Unresolved high-severity field conflicts exist
  • The case is already in MONITORING status

Why don't I see an LLM summary?

Set OPENAI_API_KEY in backend/.env and enable USE_LLM_SUMMARY=true. Without it, ForeSail uses deterministic summaries and the demo still works.

Are external events real?

By default the workflow uses configured real event sources. Overview shows the Event Mode (REAL / HYBRID / MOCK) so users can verify the active source.

My previous cases disappeared?

Case data is stored locally in the database and browser cache. Moving folders, clearing localStorage, or switching database files may remove prior cases.

Security note: Never share your .env file or commit API keys to Git.
